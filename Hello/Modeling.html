HelloFresh Recipe Score Prediction           -Krishna Punyakoti


  As HelloFresh create new recipes, it is essential to know how they will perform. A recipe score or rating is the comparative ranking that a user gives to a specific recipe. If customers like a specific recipe, it can be expected that they rate the recipe high, if the customer did not like a recipe, then they might rate it low.
  Aim of this assignment is to build a model that predicts the average recipe score of new recipes based on the available data.

In this document, I will walk you through my approach and strategy towards training a model to predict the recipe scores more accurately with better intrepretability.¶

    model = sm.OLS(y,x)
    result = model.fit()
    return result

#This function will return the calculated r2 score and rmse for given predictions and actuals
def reg_metrics(preds, actuals):
    rmse = np.sqrt(mean_squared_error(preds, actuals))
    r2 = r2_score(preds, actuals)
    return rmse, r2

Elavuation Metric:¶

I am evaluating the models' performance based on the Root Mean Squared Error. This metric can show the bigger deviations in the error clearly.

R-Squared can also help us understand the proportion of variance explained. However, if we add more variables, R-sqaured keeps increasing. Hence, we will keep this metric as a side-reference to understand the goodness of fit.

4. Final Model Selection¶

RidgeCV¶

RigdeCV has been consistently performing better with low RMSE and high R-squared statistic.

Though XGBoost has fared slightly better than linear-models, we can rely on linear models in this setting as they tend to have lesser variance than XGBoost and other non-parametric models considering that we have less number of records.

Hence, we will select RidgeCV as our final model and tune it futher to improve the performance.

We will try to optimize model performance by few techniques.
Firstly, our data is not scaled yet. During exploratory data analysis, we have noticed that log transformation made the relationship between ingredient_weight and score look more linear.

a. log transformation¶

We will try log transforming our data to see if that can reduce the RMSE further.

b. Further Scope:

This is an in-efficient model.
Since we are using only two metrics, it calculates the score based on number of ingredients and total weight of the ingredients.
If the total weight is more, the score improves and if ingredients increase, the score decreases a bit.
Hence, in an ideal scenario, if we sell 120 units of chicken breast, this model gives it the hightes score which is meaningless :).

There are many ways to improve the performance of the model further.

1. Creating new meaningful features around ingredients would help us score the recipes better
For eg: Adding categories for the ingredients such as main_ingredients, common_ingredients, premium/Exotic_ingredients can tremendously help in modeling the recipe score
This may lead to non-linear relationships but can be better explored with tree-based models

2. More Data can help in better predictions. The current dataset has only 143 training labels. With more training records, models can learn better.

3. Main Categories for dishes such as Vegeterian, Chicken, Pork/Beef can also be a helpful metric.

4. Cuisine types such as Mexican, Italian, Chinese, Asian etc can also be helpful indicators.
